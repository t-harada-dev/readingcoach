# 画面モック不足候補（2026-03-12）

SC-21 / SC-23 改修後の画面構成を基準に、追加モック要否を整理した。

## 追加候補と要否

| 候補 | 要否 | 理由 |
|---|---|---|
| 本登録完了 | 要追加 | 追加直後の導線（次アクション誘導）が現行カタログで未可視化。 |
| 空ライブラリ | 要追加 | `LibraryView` には empty 表示があるが、screen catalog の固定モックに未登録。 |
| 書影の再設定/削除 | 要追加 | SC-21 で操作ボタンを追加したため、無書影/再設定済みの両状態をモック確認したい。 |
| 現在ページ更新 | 要追加（SC-21 内シナリオ） | 進捗ON/OFFで入力可否が変わるため、同一画面内シナリオで担保する。 |
| 書籍削除 | 要仕様確認後 | 現在の UI 実装に削除導線が未定義。画面追加は仕様確定後が妥当。 |
| 読書セッション中 | 既存あり（SC-12） | 既に screen catalog 対象。追加不要。 |
| 読書完了の分岐整理 | 一部あり（SC-14/15） | 既存2画面で表現済み。読了分岐（次本選定）の可視化強化は追加検討。 |

## 最小追加案

1. `SC-20` に `empty` シナリオを追加（空ライブラリ）
2. `SC-21` に `no_cover` / `progress_off` を追加（再設定・削除、現在ページ更新の前提確認）
3. `SC-14/15` から `NextFocusNomination` への分岐を別モック化（読了分岐の明示）

## 実装反映（2026-03-12 更新）

- `SC-20`: `empty` シナリオ追加済み
- `SC-21`: `no_cover` / `cover_removed` / `rehab(progress_off相当)` 追加済み
- `SC-14/15`: `normal` / `rehab` 両シナリオで表示確認可能に拡張済み
- `SC-23`: `due` / `rehab` の両シナリオをモック表示可能
- 未着手: 「本登録完了」「書籍削除」「読了分岐（NextFocusNomination 独立モック）」は次バッチ
