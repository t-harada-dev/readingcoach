
# 積読コーチ 画面一覧 兼 画面遷移 兼 画面定義

## 0. 文書の目的

本書は、積読コーチにおける **画面一覧 / 画面遷移 / 画面定義 / ワイヤー対応ID** を 1 枚で管理するためのマスター文書である。
目的は次の 4 点。

1. 何が「画面」で、何が「surface」かを固定する
2. 状態別にどの画面が出るかを固定する
3. 各画面の役割・主 CTA・主要遷移・異常時の縮退先を固定する
4. ワイヤー / Figma frame と 1 対 1 で対応できる ID を付与する

本書は詳細 UI 仕様書ではなく、**画面設計の親台帳**である。
コピー全文、全コンポーネント仕様、全異常系、全ロジック詳細は別文書を参照する。

---

## 1. 管理ルール

## 1.1 用語

* **Screen**: アプリ本体の画面
* **Surface**: 通知、Widget、Live Activity、App Intent など、アプリ外または補助的な接点
* **State Variant**: 同じ Screen の状態違いだが、ワイヤー上は別 frame として管理するもの

## 1.2 ID 体系

* Screen ID: `SC-xx`
* Surface ID: `SF-xx`
* Wireframe ID: `WF-SC-xx` / `WF-SF-xx`

例:

* `SC-01` 初回オンボーディング_本追加
* `WF-SC-01` その画面の見た目フレーム
* `SF-02` 予定時刻通知
* `WF-SF-02` その surface の見た目フレーム

## 1.3 画面分割ルール

* 状態差により **主 CTA が変わる** 場合は別 frame とする
* 表示要素が 30% 以上変わる場合は別 frame とする
* 実装上は同一 component / screen でも、設計上は別 frame に分けてよい
* 通知 / Widget / Live Activity / App Intent は Screen と混ぜず Surface として管理する

## 1.4 この文書で扱う粒度

本書で固定するもの:

* 画面名
* 役割
* 表示条件
* 主 CTA
* 主な遷移先
* 主要表示要素
* 異常時の縮退先
* ワイヤー対応 ID

本書で固定しないもの:

* コピー全文
* 画面内の細かい余白や配色
* repository / use case の実装詳細
* Event 名の詳細定義

## 2. 全体構成

画面群は次の 5 系統で構成する。

1. オンボーディング系
2. 通常利用系
3. 再開・救済系
4. セッション / 完了後系
5. 補助 surface 系

## 6. 画面定義

## SC-01 初回オンボーディング_本追加

* 役割: 最初の 1 冊を最短で登録する
* 表示条件: 初回起動時、蔵書 0 件
* 主 CTA: 本を追加する
* 副 CTA: 手入力で追加
* 主要表示要素:

  * 見出し
  * 検索入力
  * 候補リスト
  * 手入力導線
* 遷移:

  * 候補あり -> SC-11
  * 0 件 / オフライン -> SC-10
  * 保存成功 -> SC-02
* 異常時縮退:

  * 検索失敗でも手入力へ落とす
  * 書影欠損でも登録継続

## SC-02 初回オンボーディング_時刻設定

* 役割: 日常の読書開始時刻を決める
* 表示条件: SC-01 完了後
* 主 CTA: この時間にする
* 主要表示要素:

  * 時刻ピッカー
  * 補足説明
* 遷移:

  * 完了 -> SC-03

## SC-03 初回オンボーディング_通知案内

* 役割: 通知権限を自然に案内する
* 表示条件: SC-02 完了後
* 主 CTA: 通知をオンにする
* 副 CTA: あとで
* 主要表示要素:

  * 通知の価値説明
  * 権限要求ボタン
* 遷移:

  * 許可 / スキップ -> SC-04
* 異常時縮退:

  * 拒否されても利用継続可能

## SC-04 通常ホーム

* 役割: 今日のセッションを提示し、最短で開始させる
* 表示条件: 通常時
* 主 CTA: 15分読む
* 副 CTA: 5分だけ読む
* 主要表示要素:

  * 今日のセッション
  * 次の開始時刻
  * 主 CTA
  * Ghost Link（本変更）
* 遷移:

  * 15 分開始 -> SC-12
  * 5 分開始 -> SC-24
  * 本変更 -> SC-20
  * ライブラリ -> SC-20
* 異常時縮退:

  * plan 欠損 -> 再生成して表示
  * book 参照切れ -> SC-20 へ誘導

## SC-05 通常ホーム_今日は重い

* 役割: 通常主導線を維持しつつ、心理負荷が高い日に軽量トーンで開始させる
* 表示条件: `heavy_day_signal = true`
* 主 CTA: 15分読む
* 副 CTA: 5分だけ読む
* 主要表示要素:

  * SC-04 と同等
  * 軽量トーンの補助文
  * 通常時より強く認知される 5 分導線
* 遷移:

  * 15 分開始 -> SC-12
  * 5 分開始 -> SC-24

## SC-06 Rehab ホーム

* 役割: 3 日以上未達時の再開導線
* 表示条件: `continuous_missed_days >= 3` かつ `< 7`
* 主 CTA: 1分で再点火
* 副 CTA: 5分だけ読む
* 補助要素: Rehab バナー（3分再開）
* 主要表示要素:

  * 今日のセッション
  * 回復支援コピー
  * 主 CTA
  * Rehab バナー
  * Ghost Link
* 遷移:

  * 1 分開始 -> SC-14
  * 5 分開始 -> SC-24
  * 3 分再開 -> SC-13
  * 本変更 -> SC-20
* 異常時縮退:

  * Strategic Bypass 候補がなければ既存安全本を使う

## SC-07 再開専用導線

* 役割: 7 日以上未達からの再開
* 表示条件: `continuous_missed_days >= 7`
* 主 CTA: 1分だけで再開する
* 副 CTA:

  * library を開く
  * 時間を変える
  * 今日はやめる
* 主要表示要素:

  * 再開支援コピー
  * 少数 CTA
* 遷移:

  * 1 分開始 -> SC-14
  * library -> SC-20
  * 時刻変更 -> SC-22
  * 閉じる -> 終了

## SC-08 Focus Book Picker（廃止）

* 2026-03-12 以降、SC-08 は廃止。
* 本変更導線は `SC-20 ライブラリ` に統合。
* 旧 WF-SC-08 は参照専用（新規実装・テスト対象外）。

## SC-09 本を追加する

* 役割: 検索で本を探す
* 表示条件: 本追加開始
* 主 CTA: 候補を見る
* 主要表示要素:

  * 検索バー
  * 候補リスト
  * 手入力導線
* 遷移:

  * 候補選択 -> SC-11
  * 手入力 -> SC-10
* 異常時縮退:

  * timeout / 429 / オフライン -> SC-10

## SC-10 本を追加する

* 役割: 最低限の情報で本を保存する
* 表示条件: 検索 0 件 / オフライン / API 異常
* 主 CTA: 保存する
* 入力項目:

  * タイトル（必須）
  * 著者（任意）
  * 総ページ数（任意）
  * 表紙画像を追加（任意）
* 遷移:

  * 保存成功 -> 呼び出し元へ戻る

## SC-11 本追加_候補選択

* 役割: 候補から 1 冊を決める
* 表示条件: 検索結果あり
* 主 CTA: この本を追加する
* 主要表示要素:

  * 書影または placeholder
  * タイトル / 著者 / 補助情報
* 遷移:

  * 保存成功 -> 呼び出し元へ戻る

## SC-12 Active Session_15分

* 役割: 通常セッション実行中
* 表示条件: 15 分開始後
* 主 CTA: なし
* 主要表示要素:

  * 本タイトル
  * 残り時間
  * 進行状態
* 遷移:

  * 完了 -> SC-15
* 異常時縮退:

  * クラッシュ後は復旧
  * Live Activity 失敗でも継続

## SC-13 Active Session_3分

* 役割: 3 分再開セッション実行中
* 表示条件: 3 分開始後
* 主 CTA: なし
* 主要表示要素: SC-12 と同等
* 遷移:

  * 完了 -> SC-15

## SC-14 Active Session_1分

* 役割: 1 分再点火セッション実行中（`prep_success`）
* 表示条件: 1 分開始後
* 主 CTA: なし
* 主要表示要素: SC-12 と同等
* 遷移:

  * 完了 -> SC-15
* 備考:

  * 1 分でも完了したら実績カウント対象

## SC-15 完了後サマリー_通常

* 役割: 完了直後の前進感を返し、次の行動を選ばせる
* 表示条件: セッション完了後
* 主 CTA:

  * もう 5 分やる
  * もう 15 分やる
* 補助 CTA:

  * 読了した
  * 閉じる / ホームへ戻る
* 主要表示要素:

  * 今日の達成メッセージ
  * 実行時間
  * 必要に応じ progress 表示
* 備考:

  * `少し進んだ / かなり進んだ` の自己申告は廃止
  * 一般前進はシステムメッセージで返す
  * 金額換算や asset/value 系の表示は行わない
* 遷移:

  * `もう 5 分やる` -> SC-18 または SC-24
  * `もう 15 分やる` -> SC-18 または SC-12
  * `読了した` -> SC-19
  * `閉じる / ホームへ戻る` -> SC-04 / SC-06 / SC-07
  * 初回完了かつ progress tracking 未設定時のみ SC-16 への導線を表示

## SC-16 完了後_進捗バー案内

* 役割: 初回完了後に progress tracking を opt-in で提案する
* 位置づけ: 追加仕様ではなく正式仕様
* 表示条件: 初回セッション完了後のみ
* 主 CTA: 設定する
* 副 CTA:

  * あとで
  * 使わない
* 主要表示要素:

  * 価値説明
  * 非必須であることの明示
* 遷移:

  * 設定する -> SC-17
  * あとで / 使わない -> SC-15 またはホームへ

## SC-17 完了後_進捗詳細入力

* 役割: progress bar 利用者向け初期設定
* 位置づけ: 追加仕様ではなく正式仕様
* 表示条件: SC-16 で設定するを選んだとき
* 主 CTA: 保存する
* 入力項目:

  * 総ページ数（既知なら初期値）
  * 現在ページ
* 遷移:

  * 保存成功 -> SC-15 またはホームへ
* 備考:

  * progress bar は opt-in
  * 未設定でも主導線に影響しない

## SC-18 完了後_追加セッション選択

* 役割: 続けて読む場合の追加開始（独立画面として維持。簡略UIでは SC-15 へのインライン統合可）
* 表示条件: 完了後かつ追加意欲あり
* 主 CTA:

  * もう 5 分やる
  * もう 15 分やる
* 遷移:

  * 5 分 -> SC-24
  * 15 分 -> SC-12

## SC-19 完了後_読了次本指名

* 表示条件: `finished_book`

* 主要表示要素:

  * 候補本一覧
  * library / queue 起点
* 遷移:

  * 決定 -> ホームへ

## SC-20 ライブラリ

* 役割: 蔵書一覧と詳細起点
* 表示条件: 明示的遷移時
* 主 CTA: 本を選ぶ
* 補助 CTA: 本を追加する
* 主要表示要素:

  * 蔵書一覧
  * progress bar（利用者のみ）
  * 書影または placeholder
* 遷移:

  * 本追加 -> SC-09

## SC-21 本詳細

* 役割: 保存後編集の主戦場
* 表示条件: ライブラリから選択時

  * 進捗を設定する
  * 表紙を差し替える
  * タイトルを修正する
  * 著者を修正する
  * ページ数を修正する
* 主要表示要素:

  * 書誌情報
  * progress bar
  * 現在ページ
  * 表紙画像
* 遷移:

  * Focus Book 設定 -> ホームへ
  * 進捗更新 -> 同画面反映
  * 書誌更新 -> 同画面反映

## SC-22 時刻変更

* 表示条件: 再開専用導線または設定導線
* 主 CTA: この時間にする
* 遷移:

  * 保存成功 -> 呼び出し元へ戻る

## SC-23 Due Action Sheet

* 役割: due 時点で最短開始を承諾させる
* 表示条件: 通知 body tap / due 状態で app 復帰時
* 主 CTA:

  * 開始
  * 5分だけ

  * 管理操作
* 遷移:

  * 開始 -> 状態依存の第一導線（SC-12 または SC-14）
  * 5分だけ -> SC-24
  * 30分延期 -> plan 更新

* 表示条件: 5 分開始後
* 主 CTA: なし
* 主要表示要素: SC-12 と同等
* 遷移:

  * 完了 -> SC-15

## 9. この文書の使い方

* 画面を削除する場合は、関連遷移と wireframe ID を同時に消す
