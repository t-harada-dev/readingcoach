
# 積読コーチ 画面一覧 兼 画面遷移 兼 画面定義

## 0. 文書の目的

本書は、積読コーチにおける **画面一覧 / 画面遷移 / 画面定義 / ワイヤー対応ID** を 1 枚で管理するためのマスター文書である。
目的は次の 4 点。

1. 何が「画面」で、何が「surface」かを固定する
2. 状態別にどの画面が出るかを固定する
3. 各画面の役割・主 CTA・主要遷移・異常時の縮退先を固定する
4. ワイヤー / Figma frame と 1 対 1 で対応できる ID を付与する

本書は詳細 UI 仕様書ではなく、**画面設計の親台帳**である。
コピー全文、全コンポーネント仕様、全異常系、全ロジック詳細は別文書を参照する。

---

## 1. 管理ルール

## 1.1 用語

* **Screen**: アプリ本体の画面
* **Surface**: 通知、Widget、Live Activity、App Intent など、アプリ外または補助的な接点
* **State Variant**: 同じ Screen の状態違いだが、ワイヤー上は別 frame として管理するもの

## 1.2 ID 体系

* Screen ID: `SC-xx`
* Surface ID: `SF-xx`
* Wireframe ID: `WF-SC-xx` / `WF-SF-xx`

例:

* `SC-01` 初回オンボーディング_本追加
* `WF-SC-01` その画面の見た目フレーム
* `SF-02` 予定時刻通知
* `WF-SF-02` その surface の見た目フレーム

## 1.3 画面分割ルール

* 状態差により **主 CTA が変わる** 場合は別 frame とする
* 表示要素が 30% 以上変わる場合は別 frame とする
* 実装上は同一 component / screen でも、設計上は別 frame に分けてよい
* 通知 / Widget / Live Activity / App Intent は Screen と混ぜず Surface として管理する

## 1.4 この文書で扱う粒度

本書で固定するもの:

* 画面名
* 役割
* 表示条件
* 主 CTA
* 主な遷移先
* 主要表示要素
* 異常時の縮退先
* ワイヤー対応 ID

本書で固定しないもの:

* コピー全文
* 画面内の細かい余白や配色
* repository / use case の実装詳細
* Event 名の詳細定義

## 2. 全体構成

画面群は次の 5 系統で構成する。

1. オンボーディング系
2. 通常利用系
3. 再開・救済系
4. セッション / 完了後系
5. 補助 surface 系

## 4. 状態別の出し分け

| ユーザー状態      | 第一表示                  | 補助表示                  | 主 CTA                 |
| ----------- | --------------------- | --------------------- | --------------------- |
| 初回起動・蔵書なし   | SC-01                 | なし                    | 本を追加する                |
| 初回オンボーディング中 | SC-01 → SC-02 → SC-03 | なし                    | 各ステップ完了               |
| 通常時         | SC-04 または SC-05       | SF-01 / SF-03         | 15分読む                 |
| 通常時だが今日は重い  | SC-05                 | SF-03                 | 15分読む（副 CTA: 5分だけ）    |
| 3日以上未達      | SC-06                 | SF-02 / SF-04         | 1分で再点火               |
| 7日以上未達      | SC-07                 | なし                    | 1分だけで再開する             |
| セッション中      | SC-12 / 24 / 13 / 14  | SF-05 / 09 / 06 / 07  | なし                    |
| 完了直後        | SC-15                 | 必要に応じ SC-16 / 18 / 19 | もう 5 分やる / もう 15 分やる |

## 5. 主要遷移

## 5.1 全体遷移（要約）

```text
SC-01 本追加
  -> SC-02 時刻設定
  -> SC-03 通知案内
  -> SC-04 通常ホーム

SC-04 通常ホーム
  -> SC-12 Active Session_15分
  -> SC-24 Active Session_5分
  -> SC-20 ライブラリ
  -> SC-20 ライブラリ

SC-05 通常ホーム_今日は重い
  -> SC-12 Active Session_15分
  -> SC-24 Active Session_5分
  -> SC-20 ライブラリ

SC-06 Rehab ホーム
  -> SC-14 Active Session_1分
  -> SC-24 Active Session_5分
  -> SC-13 Active Session_3分
  -> SC-20 ライブラリ
  -> SC-20 ライブラリ

SC-07 再開専用導線
  -> SC-20 ライブラリ
  -> SC-22 総合設定
  -> SC-14 Active Session_1分
  -> 閉じる

SC-09 本を追加する
  -> SC-11 候補選択
  -> SC-10 手入力

SC-11 候補選択
  -> SC-04 / SC-20 / 呼び出し元へ戻る

SC-12 / SC-13 / SC-14 Active Session
  -> SC-15 完了後サマリー
SC-24 Active Session_5分
  -> SC-15 完了後サマリー

SC-15 完了後サマリー
  -> SC-16 進捗バー案内（初回のみ）
  -> SC-18 追加セッション選択（または同画面インライン）
  -> SC-19 読了次本指名（読了時のみ）
  -> SC-04 / SC-06 / SC-07 へ戻る

SC-23 Due Action Sheet
  -> 開始（状態依存の第一導線） -> SC-12 or SC-14
  -> 5分だけ -> SC-24
  -> 30分延期 -> plan 更新して終了

SC-16 進捗バー案内
  -> SC-17 進捗詳細入力
  -> SC-15 に戻る

SC-19 読了次本指名
  -> SC-04 / SC-06 / SC-07 へ戻る
```

## 5.2 Surface 起点遷移

```text
SF-03 予定時刻通知_通常
  開始 -> reconcile -> SC-12
  5分だけ -> reconcile -> SC-24
  30分延期 -> plan 更新 -> 終了

SF-04 予定時刻通知_Rehab
  開始 -> reconcile -> SC-14
  5分だけ -> reconcile -> SC-24
  30分延期 -> plan 更新 -> 終了

SF-01 / SF-02 Widget
  主 CTA -> reconcile -> 対応する Active Session

SF-08 App Intents_開始
  読書開始 -> reconcile -> 状態依存の第一導線（SC-12 または SC-14）
  5分だけ読む -> reconcile -> SC-24
  今日の本確認 -> 最新状態表示
```

## 9. この文書の使い方

本書は次の用途に使う。

* 画面設計レビュー
* Figma frame 作成の親台帳
* 実装対象一覧
* 受入条件整理の起点
* QA テストケースの画面単位整理

運用ルール:

* 新しい画面を増やすときは、先に本書へ ID を採番する
* ワイヤーは必ず対応する `WF-*` を付ける
* 状態違いで CTA が変わるなら、別 frame を切る
* 画面を削除する場合は、関連遷移と wireframe ID を同時に消す
