# 積読コーチ UI/UX 統合指針（2026-03-09）

## 0. 目的

本指針は、`積読コーチ_論理仕様書_v1.md` の実行ロジックを UI/UX に一貫反映するための運用規約である。  
最優先は「読むかどうかの管理」ではなく、**当日達成を成立させること**。  
開始行動はそのための最小起点として最優先する。

---

## 1. 基本人格

人格は「執事」から「パフォーマンス・コーチ」へ更新する。  
ただし、執事的な丁寧さと非攻撃性は維持する。

| 要素 | 定義 |
| --- | --- |
| スタンス | 実行の伴走者。次の最小行動を提示する |
| トーン | 実務的、簡潔、非詰問、非罪悪感 |
| 目的 | 停滞の打破と再始動の成立（Execution） |
| 禁止 | 責める、理由を問い詰める、失敗ラベルを強調する |

---

## 2. ホーム画面の論理ガード

### 2.1 CTA 優先順位

* 主 CTA は常に 1 つだけ
* 通常時の主 CTA は `15分開始`
* 通常時の副 CTA は `5分だけ`
* `continuous_missed_days >= 3 && < 7` の場合、主 CTA を `1分で再点火` へ切替し、副 CTA は `5分だけ` を維持する
* 3分導線は Rehab バナー（補助 CTA）で提示する
* `continuous_missed_days >= 7` の場合は通常ホームより先に再開専用導線を出し、主 CTA は `1分だけで再開する` とする

### 2.2 Focus Book 手動変更導線

* 書影下に Ghost Link「本を切り替える」を配置
* 1 日 1 回（`MAX_DAILY_MANUAL_FOCUS_CHANGE = 1`）まで許可
* 通知アクションや DueActionSheet の主導線には置かない
* 上限到達時は Ghost Link を非表示または disabled 表示

---

## 3. Ignition と `prep_success` の意味固定

* 「1分で再点火（Ignition）」は `prep_success` 取得のための最小セッション
* 完了時は継続維持 (`continuity_streak_days`) を成立させる
* ただし読書進捗グラフの増分には使わない
* つまり「継続は守るが、進捗には嘘をつかない」を UI 表現でも厳守する

---

## 4. OS 一等地の表示運用規約

### 4.1 通知

* 本通知 CTA は `開始 / 5分だけ / 30分延期` のみ
* 文言は非詰問・非罪悪感を維持し、命令口調を避ける
* 再通知は上限を持つ（しつこく追い込まない）
* 通知からの主導線に「本を切り替える」を置かない
* `開始` は状態依存で第一導線へ map する（通常: 15分、3日以上未達: 1分、7日以上未達: 1分）

### 4.2 Widget

* 表示優先順位は「今日の1冊 -> 主 CTA -> 副 CTA」
* 管理情報（設定値、詳細統計）は表示しない
* CTA は開始系のみ（本変更・設定遷移は禁止）
* user-facing CTA は `開始 / 5分だけ` に固定する
* `救済` という語は user-facing 文言で使わない
* `開始` は状態依存で第一導線へ map する

### 4.3 Live Activity

* 表示項目は最小（本タイトル、残り時間、進行状態）
* 状態確定操作を置かない
* コア状態管理のソースにしない（表示専用）

### 4.4 DueActionSheet

* 主 CTA は常に `開始`
* 副 CTA は `5分だけ / 30分延期`
* 禁止: 本変更、Library 遷移、長文選択

### 4.5 電子書籍 deep link

* 開始記録後に best-effort で遷移する
* deep link 成否をセッション成否と混同しない

### 4.6 App Intents

* user-facing アクション名は `読書開始 / 5分だけ読む / 今日の本確認` に固定する
* `救済` という語は user-facing 文言で使わない
* `読書開始` は状態依存で第一導線へ map する

## 5. 戦略的リカバリー設計

---

### 5.1 3日連続未達時

* 主 CTA を「1分で再点火」に切替
* Rehab バナーで「3分再開」提案を提示
* 必要時は Strategic Bypass で軽量本を提示

### 5.2 7日連続未達時

* 通常ホームより先に再開専用導線を表示
* 主 CTA は `1分だけで再開する` に固定する
* 3分は主導線にしない

### 5.3 通常時 `今日は重い` variant

* 状態名: `今日は重い`
* 表示条件: `heavy_day_signal = true`
* 主 CTA は 15 分開始のまま維持する
* 副 CTA の `5分だけ` を通常時より強く認知させる
* copy は「今日は軽くでいい」「5分でも前進」系の短文を許可する

---

## 6. コミュニケーション文言ルール

### 6.1 推奨

* 「1分で再点火しましょう。ここから戻せます」
* 「準備だけでも前進です」
* 「今日は軽く進めましょう」
* 「本を机に置けたら十分です」

### 6.2 禁止

* 「なぜ読まなかったのですか？」
* 「なぜやらないのですか？」
* 「今日も失敗ですね」
* 「連続未達です」
* 罪悪感を誘導する文言全般

---

## 7. ビジュアル方針（実行特化）

* Base: オフホワイト
* Accent: エナジー・アンバー
* Border/Icon: スレートグレー
* 余計な装飾を削ったソリッドな角丸
* 見出しと主要 UI は高可読サンセリフ（SF Pro 等）
* 目的は「美しさ」より「次に押すべき行動の即時理解」

---

## 8. copy 管理規約

* 旧導線向け文言は新導線で使用しない
* 状態 enum（`plan.state` / `plan.result` / `session.mode`）ごとの copy mapping を 1 箇所に集中させる
* 同一意味の文言を画面ごとに再定義しない
* Notification / Widget / Home の文言トーンを揃える
* Widget / App Intents の user-facing 文言では `救済` を使用しない
* `prep_success` を成果誇張する文言を禁止する

---

## 9. progress tracking 規約（正式仕様）

* 初回完了後に progress tracking を任意提案してよい
* opt-in であることを明示する
* `pageCount` / `currentPage` がある場合のみ progress bar を強く描画してよい
* progress tracking 未設定でも主導線は壊さない
* 本詳細から後追い設定・更新できる
* `prep_success` と progress 表示を混線させない

---

## 10. 完了後フロー簡素化規約

* 完了画面で判断を増やしすぎない
* 前進感は copy とレイアウトで返す
* 完了後は金額換算を表示しない
* `読了した` だけを明示分岐として残す
* 一般進捗をユーザーに段階入力させない
* 追加セッション CTA（`もう 5 分やる / もう 15 分やる`）を完了画面の主役に置く
* 完了メッセージは非誇張・非説教で返す
* ユーザーに自己評価を要求しない
* 長い分析は完了直後に見せない

---

## 11. 実装時チェックリスト

* ホーム主 CTA が日次状態に応じて正しく切り替わる
* Ghost Link の表示位置と 1 日 1 回制約が守られる
* `prep_success` と読書進捗が混線しない
* 文言が非詰問・非罪悪感ルールを破っていない
* 通知 / Widget / Live Activity / DueActionSheet が許可 CTA だけを表示している
* 通常時 `15分主 / 5分副`、3日以上 `1分主 / 5分副 / 3分提案`、7日以上 `1分主` の出し分けが一致している
* 完了画面に `少し進んだ / かなり進んだ` が残っていない
* 完了画面で `もう 5 分やる / もう 15 分やる / 読了した` が表示される
